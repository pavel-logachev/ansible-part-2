# Ответ для отправки

Здравствуйте.

Выполнил домашнее задание к занятию «Ansible. Часть 2».

В репозитории приложены:

- `playbooks/01_unpack_archive.yml` - скачивает архив, создает директорию и распаковывает архив;
- `playbooks/02_install_tuned.yml` - устанавливает `tuned`, запускает сервис и включает автозагрузку;
- `playbooks/03_motd_variable.yml` - меняет `/etc/motd`, текст приветствия задан через переменную;
- `playbooks/04_motd_facts.yml` - меняет `/etc/motd`, добавляет hostname, IP-адрес и пожелание системному администратору;
- `playbooks/05_apache_role.yml` - запускает роль `apache_host_info`;
- `roles/apache_host_info` - роль для установки Apache и публикации страницы с характеристиками хоста;
- `evidence` - вывод выполнения плейбуков;
- `apache_host_info-role.zip` - архив роли.

В третьем задании роль использует Ansible facts и Jinja2-шаблон. На странице Apache выводятся CPU, RAM, размер первого HDD и IP-адрес управляемого хоста. Для перезапуска Apache реализован handler, который вызывается только при изменении шаблона. Проверка доступности сайта выполнена модулем `uri`, ожидаемый ответ: `200`.

Модули `shell` и `command` в роли не используются.

Дополнительно выполнена синтаксическая проверка всех плейбуков через `ansible-playbook --syntax-check`. Проверка страницы Apache вернула `HTTP 200`.
